前言

这是一个系列教程内容为：

  1. 学会安装相应需要的依赖软件
  2. 学会更换主题
  3. 学会基本的站点配置
  4. 知道怎么生成文章并编辑
  5. 上传你的博客并拥有一个可以访问的地址

如果你的下载速度比较慢可以选择下面的国内Git镜像进行下载

国内镜像下载：https://registry.npmmirror.com/binary.html?path=git-for-windows/

主题更换

如果觉得上面的默认主题不好看,可以更改替换主题

hexo的所有主题都在下面这个地址：

主题地址：https://hexo.io/themes/

你可能会在这个地址看到很多好看的主题，但是我们要慢慢来，有些主题的配置难度还是很高的

我们要找一个功能齐全，配置简单的主题练练手

下面是一个新手入门hexo的Next主题

预览效果：https://theme-next.org/

项目地址：https://github.com/theme-next/hexo-theme-next

配置文档：https://theme-next.org/docs/getting-started/

如你所见，其实每个主题都会包含这个三个东西

  • 预览效果: 点击链接可以预览主题部署之后的样子
  • 项目地址: 顾名思义,就是主题的项目地址
  • 配置文档: 如何配置使它成为你的个人博客的主题

下载主题

方法1

打开主题的项目地址后，点击Clone or download

然后复制https开头的地址

然后在你的博客文件夹下面打开git bash,执行下面的命令：

  • git clone ：克隆
  • 地址：下载地址
  • themes/next：是把文件夹下载到主题目录下，并把主题文件命名为next

方法2

如果你觉得比较麻烦，还有第二个办法

打开主题项目地址，然后点击Clone or download，然后点击Download ZIP

这样可以下载主题文件压缩包，前提是你已经登入了github（自己注册一下）

然后解压压缩包，把文件夹放在博客目录下的themes目录，并把文件夹命名为主题的名字

部署博客

现在我们的博客都是只能本地预览，如何将我们的上传到一个地方，然后通过域名访问呢？

方法一

需要一个服务器再买个域名绑定

方法二

github：

  • 国外网站，是英文
  • 访问速度较慢
  • 不需要实名认证
  • 域名形式可以自定义

方法三

coding：

  • 国内的平台，网站是中文
  • 访问速度快
  • 部署网站需要实名认证，只需要填写身份证号即可
  • 域名是随机给的，所以很长而且不能自定义

不喜欢实名认证还想使用自定义域名

所以接下来使用GitHub部署,使用coding部署步骤大同小异
